# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | ✅ |
| Older releases | ❌ |

Only the latest release is actively supported with security updates.

## Reporting a Vulnerability

**Do not open a public issue.** Security vulnerabilities must be reported privately.

Send an email to **security@roadmaps-faciles.fr** with:

- A description of the vulnerability
- Steps to reproduce
- Impact assessment (if possible)
- Your contact information for follow-up

## Response Timeline

| Step | Delay |
|------|-------|
| Acknowledgment | 72 hours |
| Initial assessment | 7 days |
| Fix or mitigation | 30 days (best effort) |
| Public disclosure | After fix is deployed |

## Scope

This policy covers:

- The Roadmaps Faciles application (`apps/web/`)
- Authentication and authorization flows
- Data handling and storage
- API endpoints

Out of scope:

- Third-party dependencies (report upstream)
- Social engineering attacks
- Denial of service attacks
- Issues in development/test environments only

## Disclosure Policy

We follow responsible disclosure. We ask that you:

- Give us reasonable time to fix the issue before public disclosure
- Do not access or modify other users' data
- Do not degrade the service

We will credit reporters in the security advisory (unless you prefer anonymity).
