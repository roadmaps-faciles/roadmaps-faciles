# i18n

next-intl v4 avec locale cookie-based (pas de prefix URL). Langues : fr (défaut) + en.

## Config

- `src/i18n/request.ts` : lit le cookie `NEXT_LOCALE`.
- `src/lib/utils/i18n.ts` : utilitaires + types.

## Messages

- `messages/fr.json` + `messages/en.json`.
- 24 namespaces.
- ICU plural syntax supporté.

## Navigation

`src/i18n/navigation.tsx` : re-exports simples depuis `next/link` / `next/navigation` (pas de prefix locale dans les URLs).

## Server vs client

- **Server components / actions** : `await getTranslations("namespace")` + `await getLocale()` depuis `next-intl/server`.
- **Client components** : `useTranslations("namespace")` + `useLocale()` depuis `next-intl`.

## Validation

Les schémas Zod acceptent un param `ValidationTranslator` pour traduire les messages d'erreur. Voir `src/lib/utils/zod-schema.ts`.

## Dates

- `formatDateHour(date, locale)`
- `formatRelativeDate(date, locale)`

Dans `src/lib/utils/date.ts`.

## Switch de langue

DSFR `LanguageSelect` + set du cookie + `window.location.reload()`. Code dans `src/app/LanguageSelectClient.tsx`.
